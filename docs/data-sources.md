# Data Sources

## How indexing works

Documents are split into overlapping chunks of ~600 words, tokenized, and stored in an inverted index (Windows) or Room database (Android). At query time, BM25 scoring ranks chunks by relevance and the top 12 are sent as context to the LLM.

---

## Supported file types

| Extension | Processing |
|-----------|-----------|
| `.md` | Indexed directly as text |
| `.txt` | Indexed directly as text |
| `.json` | Flattened to `key: value` lines. Arrays of objects become one line per object |
| `.pdf` | Text extracted via pdfplumber. Pre-process with `process_pdf.py` for better results |

Files over 2MB are skipped automatically. This is intentional — large raw JSON files like `forge_database.json` (8MB) would consume the entire context window. Use `strip_forge_db.py` to extract just the searchable fields.

## Adding your own sources

### Windows — drop files in docs/

Any file dropped into `docs/` or a subfolder is indexed within 2 seconds. Subfolders are supported. Source tags show the relative path from `docs/`.

To add a second folder (e.g. a separate manuals drive):
```cmd
python server.py --folder docs:default --folder "G:\Orqa Manuals":manuals --offline
```

### Android — add a source in the Sync screen

Tap ↻ → Add Source:
- **GitHub Repo:** `owner/repo` format. Syncs all `.md`, `.txt`, and useful `.json` files
- **Public URL:** direct link to a single file

### Custom JSON files

JSON files are flattened automatically. For best results, use arrays of objects with descriptive keys:

```json
[
  {
    "product": "QuadCore H7",
    "issue": "Motors 3-6 not spinning",
    "cause": "Timer5 outputs disabled when CAN bus enabled",
    "fix": "Set UAVCAN_ENABLE=0 in PX4 parameters"
  }
]
```

Each object becomes one indexed line:
```
product: QuadCore H7 | issue: Motors 3-6 not spinning | cause: Timer5...
```

## PDF processing pipeline

Raw PDF extraction produces flat text with broken line wraps and repeated headers/footers. The processing pipeline improves this significantly.

**Steps:**
1. Extract text page-by-page using pdfplumber (layout-aware)
2. Extract tables separately and convert to markdown format
3. Fix hyphenated line breaks (word-\ncontinues → wordcontinues)
4. Detect and remove repeated header/footer noise (lines appearing 3+ times)
5. Send to LLM for semantic restructuring: headers, sections, tables
6. Save as `.md` with source metadata header

**When to process:**
- Manuals with complex layouts (multi-column, sidebars)
- Datasheets with tables
- Documents with heavy headers/footers

**When to skip (`--no-restructure`):**
- Simple text-only PDFs
- When you don't want to spend LLM tokens
- Quick initial indexing — can always process later

## Cache management

### Windows
The index is rebuilt from scratch on every server start. It's in-memory only — no cache file. This means if you move or delete files from `docs/`, they disappear from the index on next restart.

### Android
Chunks are cached in Room SQLite DB at:
`/data/data/com.orqa.chat/databases/orqa_chat.db`

Management options (Sync screen):
- **Sync All** — re-fetches all enabled sources, replaces old chunks
- **Sync** (per source) — re-fetches one source
- **Clear Cache** — deletes all chunks from all sources

Clearing the cache removes all indexed content. You'll need to sync again before the chat can answer questions from docs.
